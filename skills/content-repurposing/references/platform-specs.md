# Platform Specifications

Detailed specs for each of the 10 output formats. The format-writer agent must read this file before generating any content.

---

## 1. X/Twitter Thread

**Character limit:** 280 per tweet
**Thread length:** 5-12 tweets (sweet spot: 7-9)
**Optimal tone:** Punchy, conversational, slightly provocative
**Posting cadence:** 1-3 threads per week; daily tweets acceptable
**Best times:** 8-10am, 12-1pm, 6-8pm (audience timezone)
**Algorithm notes:** Engagement in first 30 minutes determines reach. Reply to your own thread immediately with a summary or CTA. Quote-tweeting your first tweet also boosts distribution.

### Structure Template
```
Tweet 1: [HOOK — the single most scroll-stopping line]
Tweet 2: [Context — why this matters, in 1-2 sentences]
Tweet 3-N: [One key point per tweet]
Tweet N-1: [The "so what" — why the reader should care]
Tweet N: [CTA — retweet the first tweet, follow, check the link in bio]
```

### Formatting Rules
- Use line breaks within tweets for readability
- Numbers and data get their own tweets
- No hashtags in thread body; 1-2 in final tweet only
- Emojis: sparingly (1-2 per thread, if at all)
- Use "→" arrows for lists within a tweet
- Thread numbering optional (some prefer clean, some prefer "1/8" format)

## 2. LinkedIn Post

**Character limit:** 3,000 characters
**Optimal length:** 800-1,500 characters (sweet spot for engagement)
**Optimal tone:** Professional but accessible; thought leadership without jargon
**Posting cadence:** 1x per day maximum; 3-5x per week ideal
**Best times:** Tuesday-Thursday, 8-10am local time
**Algorithm notes:** Comments are weighted 10x likes. Posts that generate discussion get shown to 2nd and 3rd degree connections. First hour engagement determines total reach.

### Structure Template
```
[HOOK — first line visible before "see more"]

[Blank line]

[Context — 2-3 short paragraphs, single-line each]

[The key insight — what you learned or realized]

[Supporting evidence or example]

[Takeaway — what the reader should do with this]

[Question to drive comments]

#hashtag1 #hashtag2 #hashtag3
```

### Formatting Rules
- Single-line paragraphs with blank lines between (LinkedIn's algorithm rewards this)
- Bold key phrases with **asterisks**
- Use "→" or "•" for inline lists
- 3-5 hashtags at the very end, separated from body
- No links in the body (kills reach). Put link in first comment.
- The first line is everything. If it doesn't compel a click on "see more," nothing else matters.

## 3. Email Newsletter

**Subject line:** Under 50 characters
**Preview text:** 40-90 characters (complements, not repeats, the subject)
**Body length:** 400-700 words
**Optimal tone:** Personal, direct, one-to-one conversational
**Sending cadence:** 1-5x per week depending on list expectations
**Best days:** Tuesday, Thursday (avoid Monday inbox overload, Friday checkout)
**Open rate benchmark:** 30-45% is strong; under 20% means subject lines need work

### Structure Template
```
Subject: [Under 50 chars — curiosity or benefit driven]
Preview: [40-90 chars — adds context the subject doesn't]

[Greeting — brief, personal]

[HOOK — first sentence must earn the next sentence]

[Body — one core idea, 3 supporting points max]

[Actionable takeaway — what to do right now]

[CTA — one clear ask]

[Sign-off]

PS: [Secondary CTA or personal note — PS lines get read more than most of the email]
```

### Formatting Rules
- Bold key phrases for scanners (someone should get the gist from bold text alone)
- Short paragraphs (2-3 sentences max)
- One CTA button or link — not three
- No images unless they add information (they slow loading and trigger spam filters)
- Preview text is separate from body. Don't let it auto-generate from the first line.

## 4. LinkedIn Carousel (Text Slides)

**Slide count:** 7-10 slides
**Words per slide:** 30-50 maximum
**Optimal tone:** Educational, punchy, visual-friendly
**Posting cadence:** 1-2x per week
**Best times:** Same as LinkedIn posts
**Algorithm notes:** Carousels get 2-3x the engagement of text posts. Dwell time (how long someone spends swiping) is a strong signal.

### Structure Template
```
Slide 1: [Title — the hook. Must make someone swipe.]
Slide 2: [The problem or "what most people get wrong"]
Slides 3-8: [One point per slide, 30-50 words each]
Slide 9: [Summary or key takeaway]
Slide 10: [CTA — "Follow for more" / "Save this" / "Share with someone who needs this"]
```

### Formatting Rules
- One idea per slide, no exceptions
- Each slide must stand alone (people screenshot individual slides)
- Use numbered lists where appropriate
- Indicate suggested visual elements in [brackets]
- Write text only — the user creates the graphics in Canva or similar
- Title slide should be bold, clear, and use a number if possible ("7 Ways to..." / "The 3 Things...")

## 5. YouTube Script

**Target length:** 8-12 minutes (1,200-1,800 words)
**Optimal tone:** Spoken, conversational, personal with authority
**Publishing cadence:** 1-2 videos per week
**Best days:** Weekdays (educational content performs better Mon-Fri)
**Retention benchmark:** 50%+ average view duration is strong

### Structure Template
```
HOOK (0:00-0:15): [Most interesting claim or question — earn the click]
INTRO (0:15-0:45): [What you'll cover and why it matters]
SECTION 1 (0:45-3:00): [First key point with example]
SECTION 2 (3:00-5:30): [Second key point with example]
SECTION 3 (5:30-8:00): [Third key point with example]
SECTION 4 (8:00-10:00): [Fourth point or deeper dive — optional]
RECAP (10:00-10:30): [3 key takeaways, rapid-fire]
CTA (10:30-11:00): [Subscribe, comment, check the link]
```

### Formatting Rules
- Write for spoken cadence — short sentences, natural pauses
- Mark cues: [pause], [emphasis], [show on screen: ...], [B-roll: ...]
- Use open loops: tease upcoming points to retain viewers
- No written-English constructions ("however", "furthermore", "in conclusion")
- Include timestamps for the description
- Write a suggested title (under 60 chars) and description (2-3 sentences)

## 6. Short-Form Video Script (TikTok/Reels/Shorts)

**Target length:** 30-60 seconds (75-150 words)
**Optimal tone:** Urgent, surprising, direct — imagine a stranger scrolling past
**Publishing cadence:** 1-3x per day (algorithm rewards volume)
**Best times:** Algorithm-driven, not time-dependent
**Hook window:** First 3 seconds determine if someone watches

### Structure Template
```
HOOK (0-3 sec): [Single most provocative sentence]
BODY (3-45 sec): [One idea, explained simply and fast]
CTA (45-60 sec): [Follow for more / Comment your answer / Save this]
```

### Formatting Rules
- This is NOT a condensed YouTube script. It's a completely different format.
- One idea, one angle, maximum speed
- Mark pacing: [quick cut], [text overlay: ...], [pause for emphasis]
- The hook must work without any context
- Conversational: explain it like you're telling a friend in an elevator
- Suggest text overlays that reinforce the spoken words

## 7. Blog Summary / SEO Snippet

**Length:** 200-400 words
**Optimal tone:** Clear, informative, keyword-aware
**Use cases:** Guest posts, Medium syndication, roundup contributions, meta descriptions
**SEO notes:** Include 2-3 target keywords naturally

### Structure Template
```
[Hook paragraph — 2-3 sentences summarizing the core insight]

Key takeaways:
• [Point 1]
• [Point 2]
• [Point 3]
• [Point 4]

[Closing paragraph with link to the full piece]

Meta description: [Under 160 characters]
Target keywords: [2-3 keywords]
```

### Formatting Rules
- Must be useful standalone but clearly signal deeper content exists
- Front-load keywords in the first paragraph
- Bullet points for scannability
- Include a suggested meta description
- No fluff — every sentence must carry information

## 8. Quote Graphics (Text for Canva)

**Quote length:** 10-25 words per quote
**Number of quotes:** 5-8 per source
**Optimal tone:** Authoritative, memorable, standalone
**Platforms:** Instagram, Pinterest, X, LinkedIn
**Design notes:** Text must be readable at phone-screen size

### Structure Template (per quote)
```
Quote: "[The quote text]"
Emphasis: [Word(s) to highlight or bold]
Style: [Light background / Dark background / Gradient]
Attribution: [Author name — Source title]
Platform fit: [Which platforms this works best on]
```

### Selection Rules
- Quotes must be verbatim from the source (or very close paraphrases if the original is too long)
- Select for: surprise, insight, contrarian take, emotional resonance
- Each quote must make sense without context
- Avoid quotes that start with "I" (less shareable)
- Mix lengths: some short (10-12 words), some medium (18-25 words)

## 9. Podcast Talking Points

**Length:** 8-12 bullet points with 2-3 sub-bullets each
**Optimal tone:** Conversational, exploratory, story-ready
**Use cases:** Solo episodes, guest appearances, interview prep
**Note:** This is NOT a script. Podcasters sound terrible reading scripts.

### Structure Template
```
OPENING
• How to introduce the topic: [hook for listeners]
  - [Anecdote or example to open with]
  - [Why this matters to the listener right now]

POINT 1
• [Main point]
  - [Supporting detail or example]
  - [Potential tangent: ...]

POINT 2
• [Main point]
  - [Supporting detail]
  - [Real-world example]

[Continue for 3-5 main points]

CLOSING
• The one takeaway: [single actionable thing]
  - [How to start]
  - [Where to go deeper]
```

### Formatting Rules
- Bullets, not sentences. Give the host anchor points, not words to read.
- Include suggested anecdotes and analogies that work well spoken aloud
- Mark potential tangent topics: [tangent: this could spin into...]
- Tone guidance note at the top: "Talk about this like you're explaining it to a curious friend at dinner"
- Include a suggested episode title

## 10. Community Post (Skool/Discord/Facebook Groups)

**Length:** 100-250 words
**Optimal tone:** Peer-to-peer, curious, discussion-starting
**Posting cadence:** 2-3x per week
**Engagement key:** The question at the end matters more than the content
**Algorithm notes:** Groups suppress posts with links. Put links in a comment.

### Structure Template
```
[HOOK — observation or surprising claim]

[Brief context — 2-3 sentences max]

[The insight or take — what you learned or noticed]

[Specific, answerable question for the community]
```

### Formatting Rules
- No links in the body (add them in a reply/comment)
- The question must be specific: "Has anyone else found that..." not "Thoughts?"
- Peer-to-peer tone — not teacher-to-student, not brand-to-audience
- Reference shared experiences: "If you've tried..." / "Anyone else noticed..."
- Keep it short. Community members scroll fast. Under 200 words is ideal.
- Don't be promotional. Share genuinely, ask genuinely.
